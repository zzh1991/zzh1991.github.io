知识管理

我们每天都在接触各种各样的信息，并从中提取出自己所需的知识，那下面就来聊聊如何利用工具来管理知识。

笔记类工具

为知笔记: 跟 Evernote 和有道云笔记是很类似的

  • 本人已经付费两年，从免费版用到订阅版，一年 60 元，并不是非常贵
  • 支持 Markdown 模式，版本控制，跨平台同步，体验就好
  • 目前用来记录一些比较琐细的 idea 和知识点，后期再作统一处理
缺点
  • Markdown 的阅读模式体验还是很不错的，但是编辑的时候不是非常有效率
  • 富文本的排版比较麻烦，所以一般都采用 Markdown 书写

语雀: 阿里旗下出品的知识管理分享产品

  • 网址：www.yuque.com
  • 从内测开始使用，现在已经开放注册了
  • 自动排版：富文本排版，样式好看
  • 知识库管理：类似笔记库
  • 目录索引：与为知笔记一样具有协作分享功能
  • 目前主要用其记录比较详细，内容比较多的笔记
缺点
  • 目前没有移动端，只有网页端，手机只能打开浏览器进行查看

Notion: 颠覆式笔记应用

  • 拖拽式组合网页：随意插入想要的组件
  • 笔记之间可以相互链接，类似网页
  • 谷歌账号可以直接登入，不需要注册账号
  • 支持嵌入网页内容、媒体，功能十分强大
  • iPhone，Andriod，Windows，Mac，网页端
  • 目前我主要用其记录一些网页的链接和图片
缺点
  • 高级功能需要付费，而且价格不菲，4 美元 / 月
  • 手机 APP 和网页端只能进行 1000 次的操作，超出就需要付费才能进行修改，现在支持删除已有的条目，然后继续使用
  • 手机上的操作并不会影响数目，所以尽量在手机上操作（目前已经修改规则了）

Airtable & Trello

  • 主要用于一些任务的记录与跟踪
  • Airtable 可以集成 Slack 和 Trello（借助 Zapier）
缺点
  • Airtable 免费版不能使用一些 Block 可视化插件
  • Airtable 与 Trello 的互动比较少

日历工具 Sunsama

  • 对接 Google 日历
  • 集成 Slack、Trello
  • 一个 channel 跟踪一个 GitHub 仓库的变更
缺点
  • 只有网页版和 Mac 端应用，没有手机应用
  • 一个 channel 只能设置一个 GitHub 仓库，不能设置成多个
  • 免费版也有操作的次数限制

todo: 自己实现的 todo 管理工具

功能（目前还是比较基础的）

  1. 提供编辑器记录详细信息，支持 Markdown 语法
  2. 比较适合网页版操作，目前并没有适配移动端
  3. 详细的时间记录
  4. 支持排序功能，后续会加入搜索、过滤功能
